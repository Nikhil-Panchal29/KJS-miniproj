EXPERIMENT No 1:Poset


INSTRUCTIONS:

1) Interface Overview:
a) The main window is simple and contains the following components:
i) Instruction Label: Provides instructions to enter the relation in the form {(x,y),(x,y),...}.
ii) Relation Input Field: Enter the relation in the specified format.
iii) Check Button: Click this button to check if the entered relation is a poset.
iv) Result Label: Displays the result of the poset check.


2) Entering the Relation:
a) Enter the relation in the text field following the format {(x1,y1),(x2,y2),...}.
b) For example, a relation {(1,1),(1,2),(2,2)} can be entered.


3) Checking for Poset:
a) After entering the relation, click the "Check" button.
b) The Result Label will be updated with either "The relation is a poset" or "The relation is not a poset" based on the properties of reflexivity, antisymmetry, and transitivity.


4) Viewing Results:
a) The Result Label will display the outcome of the poset check.


5) Example:
a) For example, if you enter the relation {(1,1),(1,2),(2,2)}, the Result Label will display "The relation is a poset" because it satisfies the properties of reflexivity, antisymmetry, and transitivity.


6) Closing the Application:
a) Close the application window when you are done using it.


Notes:
a) The simulator uses the concepts of reflexivity, antisymmetry, and transitivity to determine if the entered relation forms a poset.
b) The relation is entered in a specific format, and it's important to include parentheses and commas as specified.
c) The simulator uses a Pair class to represent ordered pairs and override the equals and hashCode methods for set operations.
